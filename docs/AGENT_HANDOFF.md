# AGENT_HANDOFF.md - AI Agent 交接协议

本项目可能被多个 AI Agent 接力开发，本文件定义交接规则。

## 交接时机

- 完成一个独立功能模块后
- 完成一个阶段性目标后
- 需要换人继续开发时

## 交接内容

### 必须交接

1. **当前进度**：完成了什么、正在做什么
2. **下一步计划**：待完成的任务列表
3. **已知问题**：卡点、风险、未解决的问题
4. **代码位置**：关键代码的文件位置
5. **验证状态**：已通过哪些检查

### 交接格式

```
## 当前状态

- 进度：已完成 X / 总共 Y
- 阻塞：无 / 有（说明）

## 完成的工作

1. 功能 A（文件位置）
2. 功能 B（文件位置）

## 待完成

1. 功能 C
2. 功能 D

## 验证

- build: 通过
- check: 通过
- self-test: 通过
- preflight: 通过 / 有警告

## 注意事项

- 某部分代码需要特别注意
- 某功能尚未完全测试
```

## 接收方职责

1. 先读 AGENTS.md 了解项目规则
2. 再读本文件了解交接内容
3. 运行验证确保环境正常
4. 继续完成待办任务

## 禁止事项

- 不交接未验证的代码
- 不交接有已知问题的代码
- 不交接未完成的功能（除非明确标记）
