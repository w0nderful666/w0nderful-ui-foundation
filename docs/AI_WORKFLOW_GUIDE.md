# AI 项目工作流 v1（使用手册）

这套工作流解决三件事：

1. 新项目能快速初始化
2. 旧项目能快速恢复上下文
3. 对话变长/模型降智后能靠 `AGENT_HANDOFF.md` 快速重启

核心分工：

- `GLOBAL_SKILL.md`：跨项目通用规则（默认只读）
- `AGENT_HANDOFF.md`：项目专属事实与约束（可更新，但要克制）
- 高智能体：分析、拆任务、写执行提示词、辅助验收
- 低智能体/执行模型：只执行一个小任务 + 跑验证 + 报告结果

---

## 1) 新项目怎么初始化

1. 读取 `GLOBAL_SKILL.md`
2. 在项目根目录创建 `AGENT_HANDOFF.md`（复制 `AGENT_HANDOFF_TEMPLATE.md`，只填**事实**）
3. 填好最少三块：技术栈、常用命令、测试门禁/回归清单
4. 初始化完成后，再开始第一轮小任务

## 2) 旧项目已有 `AGENT_HANDOFF.md` 怎么继续

1. 读取 `GLOBAL_SKILL.md`
2. 读取 `AGENT_HANDOFF.md`（恢复上下文与约束）
3. 只读取与本轮目标相关的少量文件
4. 开始拆任务与执行

## 3) 旧项目没有 `AGENT_HANDOFF.md` 怎么补

1. 读取 `GLOBAL_SKILL.md`
2. 只扫描关键入口文件与构建/测试脚本（不要全仓扫）
3. 生成最小可用 `AGENT_HANDOFF.md`（先写事实与门禁，不写流水账）
4. 再开始具体任务

---

## 4) 一轮任务怎么开始（标准流程）

1. 用户提出需求
2. 高智能体分析需求与约束
3. 高智能体拆成小任务（每次只选一个做）
4. 高智能体用 `TASK_PROMPT_TEMPLATE.md` 生成执行提示词
5. 执行模型只做这一个小任务
6. 执行模型跑测试/验证（按提示词指定的命令）
7. 执行模型按模板输出结果，并说明是否建议更新 `AGENT_HANDOFF.md`

## 5) 怎么让高智能体拆任务

拆分准则：

- 一个任务 = 一个可验收的最小变更
- 每个任务都有：完成标准 + 验证方式 + 影响范围
- 遇到范围不清或风险大：先给选项，让用户拍板

## 6) 怎么让低智能体执行

把填好的 `TASK_PROMPT_TEMPLATE.md` 内容原样发给执行模型，并强调：

- 不扩展范围
- 只改白名单文件
- 不改 `GLOBAL_SKILL.md`
- 不编造测试结果
- 失败就报告真实报错与下一步建议

## 7) 什么时候更新 `AGENT_HANDOFF.md`

只在出现**长期有效信息**时更新，例如：

- 新增/变更了常用命令、测试门禁、发布流程
- 发现了新的“长期坑点”或禁止破坏点
- 确认了新的稳定基线（通过了哪些验证）
- 新增了重要的目录/关键文件职责说明

不要更新的情况：

- 只是一次性的排错过程
- 只是聊天过程中的临时想法
- 没有新增事实/约束/基线

## 8) 对话变长降智后怎么办

1. 新建对话
2. 先读 `GLOBAL_SKILL.md`
3. 再读 `AGENT_HANDOFF.md`
4. 只读与目标相关文件
5. 继续按“一轮一个小任务”的节奏推进

## 9) 项目结束后怎么提炼进 `GLOBAL_SKILL.md`

只有当经验“跨项目可复用”时才提炼进去，例如：

- 通用的任务拆分方法、验收口径、发布门禁
- 通用的质量规则（例如：不编造测试、变更后必须可验证）

项目私有内容（路径、线上地址、特定业务约束）不要写进 `GLOBAL_SKILL.md`。
