# SLOW_QUALITY_AGENT.md - 慢质量模式

本文件定义项目中的慢质量工作模式。

## 何时进入

- 修复 bug
- 处理复杂功能
- 改动影响多个模块
- 遇到不确定问题

## 工作流程

### 1. Read - 读取

- 读项目规则文件
- 读相关源码
- 读测试文件
- 读样式文件

### 2. Understand - 理解

- 说明当前实现
- 说明数据流
- 说明 UI 结构
- 说明样式来源

### 3. Diagnose - 诊断

- 判断根因
- 不能确认时扩大阅读范围

### 4. Plan - 计划

- 给出最小修改方案
- 评估影响范围

### 5. Patch - 修补

- 小范围修改
- 复用已有结构

### 6. Verify - 验证

- 运行 build
- 运行 check
- 运行 self-test
- 运行 preflight

### 7. Review - 审查

- 自查是否破坏项目一致性
- 确保不引入第二套风格

## 原则

- 宁愿慢，不要乱
- 宁愿小步修，不要大重构
- 宁愿先分析，不要直接瞎改
- 宁愿报告不确定点，不要猜测式修改
- 项目一致性优先于修复速度

## 验证清单

- [ ] build 通过
- [ ] check 通过
- [ ] self-test 通过
- [ ] preflight 通过
- [ ] 未破坏已有功能
- [ ] 未改变 UI 风格
- [ ] 未引入第二套实现
