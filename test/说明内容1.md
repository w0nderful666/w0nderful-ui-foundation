# 一、重装后完整初始化脚本

这个脚本目标：

```text
1. 开机自检
2. 换 USTC Debian 源
3. 安装基础工具
4. 安装 Cloudflare WARP
5. 注册 WARP
6. 设置 WARP Local Proxy 模式
7. 启动 127.0.0.1:40000 本地代理
8. 测试 warp=on
9. 写入 /root/AGENT_HANDOFF.md
```

> 适用环境：Debian 12 bookworm 纯 IPv6 小主机。
> 不适合：想让 WARP 接管整机默认路由的场景。
> 重点：**不会执行 `warp-cli mode warp`，不会主动配置全局 IPv4。**

直接新建脚本：

````bash
cat > /root/setup-warp-proxy-v6.sh <<'SCRIPT'
#!/usr/bin/env bash
set -euo pipefail

PROXY_PORT="40000"
PROXY_ADDR="127.0.0.1:${PROXY_PORT}"
TS="$(date +%F-%H%M%S)"

echo "============================================================"
echo "  Pure IPv6 Debian WARP Local Proxy Setup"
echo "  Host: $(hostname)"
echo "  Time: $(date)"
echo "============================================================"
echo

if [ "$(id -u)" -ne 0 ]; then
  echo "ERROR: 请使用 root 执行本脚本。"
  exit 1
fi

echo "[1/9] 开机自检"
echo "---- hostname ----"
hostname || true

echo
echo "---- os-release ----"
cat /etc/os-release || true

echo
echo "---- IPv6 route ----"
ip -6 route || true

echo
echo "---- IPv6 connectivity test ----"
curl -6 -m 10 ip.sb || true
echo

echo
echo "---- IPv4 connectivity test before WARP, failure is normal on pure IPv6 ----"
curl -4 -m 10 ip.sb || true
echo

echo
echo "[2/9] 换 USTC Debian bookworm 国内源"
if [ -f /etc/apt/sources.list ]; then
  cp -a /etc/apt/sources.list "/etc/apt/sources.list.bak.${TS}"
fi

cat > /etc/apt/sources.list <<'EOF'
deb https://mirrors.ustc.edu.cn/debian/ bookworm main contrib non-free non-free-firmware
deb https://mirrors.ustc.edu.cn/debian/ bookworm-updates main contrib non-free non-free-firmware
deb https://mirrors.ustc.edu.cn/debian-security bookworm-security main contrib non-free non-free-firmware
EOF

apt update

echo
echo "[3/9] 安装基础工具"
apt install -y curl wget ca-certificates gpg lsb-release vim nano iproute2 net-tools dnsutils

echo
echo "[4/9] 添加 Cloudflare WARP 软件源"
curl -fsSL https://pkg.cloudflareclient.com/pubkey.gpg \
  | gpg --yes --dearmor \
  --output /usr/share/keyrings/cloudflare-warp-archive-keyring.gpg

cat > /etc/apt/sources.list.d/cloudflare-client.list <<'EOF'
deb [signed-by=/usr/share/keyrings/cloudflare-warp-archive-keyring.gpg] https://pkg.cloudflareclient.com/ bookworm main
EOF

apt update

echo
echo "[5/9] 安装 Cloudflare WARP"
apt install -y cloudflare-warp
systemctl enable --now warp-svc
sleep 2

echo
echo "---- warp-svc status ----"
systemctl status warp-svc --no-pager || true

echo
echo "[6/9] 检查 warp-cli 是否支持 proxy 模式"
warp-cli --help | head -n 40 || true
echo
warp-cli mode --help || true
echo
warp-cli proxy --help || true

echo
echo "[7/9] 注册 WARP"
echo "说明：这里使用 --accept-tos 自动接受 Cloudflare WARP 客户端条款。"
echo "如果已经注册过，这一步可能提示已存在或失败；新系统通常会 Success。"

if ! warp-cli --accept-tos registration new; then
  echo "WARN: registration new 未返回成功。可能是已经注册过，也可能是网络问题。继续尝试检查状态。"
fi

warp-cli status || true

echo
echo "[8/9] 设置 WARP Local Proxy 模式"
echo "关键：只使用 proxy 模式，不使用 warp / warp+doh / tunnel_only。"

warp-cli mode proxy
warp-cli proxy port "${PROXY_PORT}"

echo
echo "连接 WARP Proxy..."
warp-cli connect

echo
echo "等待本地代理可用：${PROXY_ADDR}"
sleep 3

echo
echo "---- warp-cli status ----"
warp-cli status || true

echo
echo "---- listen check ----"
ss -tulnp | grep "${PROXY_PORT}" || true

echo
echo "---- WARP proxy test ----"
OK=0
for i in $(seq 1 12); do
  echo "Test attempt ${i}/12..."
  if curl -x "socks5h://${PROXY_ADDR}" -m 20 https://www.cloudflare.com/cdn-cgi/trace | grep -E 'ip=|warp='; then
    OK=1
    break
  fi
  sleep 3
done

if [ "$OK" -ne 1 ]; then
  echo
  echo "ERROR: WARP 代理测试失败。"
  echo "请检查："
  echo "  warp-cli status"
  echo "  journalctl -u warp-svc -n 80 --no-pager"
  exit 1
fi

echo
echo "---- proxy IP test ----"
curl -x "socks5h://${PROXY_ADDR}" -m 20 https://ip.sb || true
echo

echo
echo "[9/9] 写入 /root/AGENT_HANDOFF.md"
if [ -f /root/AGENT_HANDOFF.md ]; then
  cp -a /root/AGENT_HANDOFF.md "/root/AGENT_HANDOFF.md.bak.${TS}"
fi

cat > /root/AGENT_HANDOFF.md <<'HANDOFF'
# Server Agent Handoff - ser608564523726 / 纯 IPv6 襄阳

## 机器定位

这台机器是纯 IPv6 Debian 12 小主机。

核心定位：

- 原生 IPv6：用于 SSH 管理服务器。
- Cloudflare WARP：只用于本地代理访问 IPv4 / 受限网络。
- 不追求整机全局 IPv4。
- 不允许让 WARP 接管默认路由。

## 当前 WARP 方案

本机使用 Cloudflare WARP Local Proxy 模式：

```text
127.0.0.1:40000
````

已验证命令：

```bash
curl -x socks5h://127.0.0.1:40000 -m 20 https://www.cloudflare.com/cdn-cgi/trace | grep -E 'ip=|warp='
curl -x socks5h://127.0.0.1:40000 -m 20 https://ip.sb
```

如果输出包含：

```text
warp=on
```

说明 WARP 代理可用。

## 严禁执行

不要执行以下命令：

```bash
warp-cli mode warp
warp-cli mode warp+doh
warp-cli mode tunnel_only
```

不要把这台纯 IPv6 主机改造成“整机全局 IPv4”模式。

不要用下面这个命令判断 WARP 是否可用：

```bash
curl -4 ip.sb
```

`curl -4 ip.sb` 失败不代表 WARP 坏了，因为本机设计目标不是全局 IPv4，而是通过本地代理访问 IPv4 网络。

## WARP 状态检查

```bash
systemctl status warp-svc --no-pager
warp-cli status
ss -tulnp | grep 40000 || true
```

## WARP 代理测试

优先使用：

```bash
curl -x socks5h://127.0.0.1:40000 -m 20 https://www.cloudflare.com/cdn-cgi/trace | grep -E 'ip=|warp='
curl -x socks5h://127.0.0.1:40000 -m 20 https://ip.sb
```

看到：

```text
warp=on
```

表示 WARP 代理工作正常。

## curl 使用代理

```bash
curl -x socks5h://127.0.0.1:40000 https://ip.sb
```

## 当前 Shell 临时使用代理

如果 AI、OpenCode、Claude Code、git、npm、curl 需要访问 IPv4 或国外网络，先在当前 shell 执行：

```bash
export ALL_PROXY=socks5h://127.0.0.1:40000
export HTTP_PROXY=http://127.0.0.1:40000
export HTTPS_PROXY=http://127.0.0.1:40000
export NO_PROXY=localhost,127.0.0.1,::1
```

取消代理：

```bash
unset ALL_PROXY HTTP_PROXY HTTPS_PROXY NO_PROXY
```

## git 使用代理

临时 clone：

```bash
git -c http.proxy=socks5h://127.0.0.1:40000 clone <repo-url>
```

全局设置：

```bash
git config --global http.proxy socks5h://127.0.0.1:40000
git config --global https.proxy socks5h://127.0.0.1:40000
```

取消全局代理：

```bash
git config --global --unset http.proxy || true
git config --global --unset https.proxy || true
```

## npm 使用代理

设置：

```bash
npm config set proxy http://127.0.0.1:40000
npm config set https-proxy http://127.0.0.1:40000
```

取消：

```bash
npm config delete proxy
npm config delete https-proxy
```

## apt 注意事项

不要默认给 apt 写死代理。

本机已使用 USTC Debian 国内源，通常 apt 不需要走 WARP。

如果 apt 临时需要代理，可以单次使用：

```bash
http_proxy=http://127.0.0.1:40000 https_proxy=http://127.0.0.1:40000 apt update
```

## AI / Agent 使用规则

后续 AI 在这台机器上工作时必须遵守：

1. 需要访问 IPv4 / GitHub / npm / API 时，优先使用 `127.0.0.1:40000` 代理。
2. 不要修改系统默认路由。
3. 不要切换 WARP 到 `warp`、`warp+doh`、`tunnel_only` 模式。
4. 不要把代理写死到系统级服务，除非用户明确要求。
5. 如果网络不通，先测试 WARP 代理，而不是重装系统。
6. 不要把 `curl -4 ip.sb` 失败当成系统故障。

## 快速自检命令

```bash

```

## 本机最终目标

正确目标：

```text
需要 IPv4 的程序可以通过本地代理访问网络。
SSH 和系统默认 IPv6 路由保持稳定。
```

错误目标：

```text
整台纯 IPv6 机器获得全局 IPv4 默认路由。
```

HANDOFF

echo
echo "============================================================"
echo "完成。"
echo
echo "请重点确认下面输出里有 warp=on："
curl -x "socks5h://${PROXY_ADDR}" -m 20 [https://www.cloudflare.com/cdn-cgi/trace](https://www.cloudflare.com/cdn-cgi/trace) | grep -E 'ip=|warp=' || true
echo
echo "Handoff 文件位置：/root/AGENT_HANDOFF.md"
echo "============================================================"
SCRIPT

chmod +x /root/setup-warp-proxy-v6.sh

````

执行：

```bash
/root/setup-warp-proxy-v6.sh
````
